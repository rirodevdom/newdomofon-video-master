# Managed camera access tokens

Managed camera tokens replace ad-hoc token generation from the camera-link button.

## Model

1. A super administrator creates a named token in **Administration → Tokens**.
2. The token can allow video (`camera`), events (`events`), or both.
3. In **Camera links**, the administrator selects an existing token and clicks **Open links**.
4. Master creates an idempotent token-to-camera assignment and returns URLs that contain the selected token.
5. The public SmartYard gateway sends the token and stream name to master.
6. Master validates the managed token, assignment, status, generation, expiry and scopes, then mints a short-lived node token.

The stable managed token is never accepted directly by a node.

## Revocation

- Disable: all links using the token stop resolving until it is enabled again.
- Rotate: generation increases; every previously copied link becomes invalid immediately.
- Delete: token and all camera assignments are removed.
- Expire: links stop resolving after `expires_at`.

Previously issued legacy camera tokens remain accepted by the internal resolver for compatibility, but the administration UI no longer creates them.

## Database

Migration `090_managed_camera_tokens.sql` creates:

- `managed_camera_tokens`;
- `managed_camera_token_cameras`.

A token is not allowed to access a camera until the assignment row exists. Opening links with a selected token creates that assignment with `ON CONFLICT DO NOTHING`.

## Signing secret

Managed tokens use:

```env
MANAGED_CAMERA_TOKEN_SECRET=<long-random-secret>
```

When the variable is not configured, `JWT_SECRET` is used. Changing the effective secret invalidates all managed camera links. Use a stable value in production and back it up with the rest of the master secrets.

## API

```text
GET    /api/tokens/managed-camera-tokens
POST   /api/tokens/managed-camera-tokens
PATCH  /api/tokens/managed-camera-tokens/:tokenId
POST   /api/tokens/managed-camera-tokens/:tokenId/rotate
DELETE /api/tokens/managed-camera-tokens/:tokenId
DELETE /api/tokens/managed-camera-tokens/:tokenId/cameras/:cameraId
POST   /api/tokens/camera-links/:cameraId
POST   /api/tokens/smartyard-links/:cameraId
```

Both camera-link endpoints require:

```json
{
  "managed_token_id": "uuid"
}
```

No new token is generated by those endpoints.
